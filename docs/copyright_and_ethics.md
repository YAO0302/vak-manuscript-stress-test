# Copyright and Ethics

This document defines the copyright, academic integrity, and ethical boundaries of this project.

The project is designed to support responsible academic writing, journal style analysis, manuscript review, and revision.

It must not be used for plagiarism, ghostwriting, citation fabrication, copyright violation, or deceptive academic practice.

## 1. Core principle

This project helps users analyze high-level academic writing patterns.

It does not authorize copying, redistributing, or imitating copyrighted journal articles.

The project should extract:

* article structure;
* abstract patterns;
* introduction logic;
* methodology language;
* argumentation models;
* citation behavior;
* general academic phrases;
* conclusion patterns.

The project must not extract or reuse:

* original paragraphs;
* distinctive sentences;
* unique metaphors;
* specific arguments from another author;
* unpublished materials without permission;
* copyrighted PDF files committed to a public repository.

## 2. Use of target journal articles

Users may analyze articles only if they have lawful access to them.

Examples of lawful access may include:

* open access journal websites;
* university library access;
* author-provided copies;
* institutional databases;
* personal research access.

Users must not upload copyrighted articles into this public repository.

Target journal articles should be used only in a local workspace or private AI workspace for style analysis.

## 3. No redistribution of journal articles

This repository must not contain:

* downloaded journal PDFs;
* copyrighted article scans;
* copied full texts;
* paywalled articles;
* private manuscripts;
* unpublished peer review materials;
* student papers without permission;
* supervisor comments without permission.

The following folders and files should remain private and should not be committed:

* samples/
* uploads/
* articles/
* pdfs/
* source_articles/
* drafts/
* private/
* thesis/
* dissertation/
* *.pdf
* *.doc
* *.docx

These file types are blocked in `.gitignore`.

## 4. Style analysis boundary

The project may analyze style at a high level.

Allowed:

* identifying common article structures;
* describing abstract patterns;
* summarizing introduction logic;
* identifying common rhetorical moves;
* extracting short general academic formulas;
* describing citation density;
* describing conclusion structure;
* creating a reusable journal style profile.

Not allowed:

* copying paragraphs from published articles;
* rewriting another author's article as if it were original;
* imitating one author's distinctive style;
* reproducing protected expression;
* using uploaded articles as hidden source material for plagiarism;
* generating text that closely follows one specific article.

## 5. Reference integrity

This project must not generate fake references.

The system must not:

* invent authors;
* invent article titles;
* invent DOI numbers;
* invent journal issues;
* invent page ranges;
* fabricate database records;
* silently correct suspicious references without verification.

Every reference should satisfy three conditions before submission:

1. The source is real.
2. The bibliographic data are correct.
3. The source supports the claim it is attached to.

If a source cannot be verified, it must be marked as requiring external verification.

## 6. AI-generated text

AI-generated text must be treated as draft material.

Users are responsible for:

* checking factual accuracy;
* verifying all references;
* confirming that claims are supported;
* revising the text intellectually;
* complying with journal rules;
* complying with university and publisher policies;
* disclosing AI assistance when required.

AI output should not be submitted without human review.

## 7. No ghostwriting policy

This project is not intended to replace the author.

The author must remain responsible for:

* research design;
* source selection;
* data or material analysis;
* argument development;
* interpretation;
* final manuscript content;
* ethical compliance.

The project may help revise, audit, and stress-test a manuscript, but it should not be used to submit work that the author does not understand or cannot defend.

## 8. Peer review simulation boundary

The hostile peer review module is a revision tool.

It is not a real journal review.

It does not guarantee publication.

It should be used to identify:

* weak research questions;
* method-question mismatch;
* logical breaks;
* unsupported claims;
* weak novelty;
* reference risks;
* structural problems.

Final judgment remains with human authors, supervisors, editors, and reviewers.

## 9. Responsible use with ВАК journals

For Russian ВАК journal submissions, users should check:

* target journal requirements;
* author guidelines;
* bibliography rules;
* originality and anti-plagiarism requirements;
* AI-use policies, if available;
* university rules;
* field-specific academic standards.

This project does not replace official ВАК, journal, university, or dissertation council requirements.

## 10. Practical safety rules

Before using this project, follow these rules:

1. Do not commit journal PDFs to GitHub.
2. Do not commit private manuscript drafts to GitHub.
3. Do not use AI-generated references without verification.
4. Do not copy text from uploaded articles.
5. Do not imitate an individual author's distinctive style.
6. Do not submit AI output without human revision.
7. Do not use citations that do not support the claim.
8. Do not hide weak logic behind defensive language.
9. Do not use the project to bypass academic responsibility.
10. Keep source articles and drafts private.

## 11. Disclaimer

This project is an academic writing and manuscript review aid.

It does not provide legal advice.

It does not provide official journal approval.

It does not guarantee publication.

It does not replace supervisors, editors, reviewers, university rules, publisher rules, or legal requirements.

Users are responsible for how they use this project.
