# Limitations

This document explains the limitations of the VAK Manuscript Stress Test project.

The project is a writing, auditing, and review-support workflow. It is not a guarantee of publication, factual accuracy, reference validity, or academic approval.

## 1. This project does not guarantee publication

The project can help improve a manuscript before submission.

It can identify:

* weak research questions;
* method-question mismatch;
* logical breaks;
* weak novelty;
* unsupported claims;
* suspicious references;
* unclear structure;
* excessive defensive writing.

However, it cannot guarantee that a journal will accept the manuscript.

Final publication decisions are made by editors, reviewers, editorial boards, and publishers.

## 2. AI cannot fully verify references without evidence

The reference audit module can detect many risks, including:

* missing bibliographic data;
* DOI mismatch;
* suspicious source patterns;
* uncited bibliography entries;
* in-text citations missing from the bibliography;
* claims that are not clearly supported.

However, AI cannot fully prove that a reference is real unless it has access to reliable evidence.

Reliable evidence may include:

* DOI records;
* publisher pages;
* journal websites;
* eLIBRARY records;
* CyberLeninka pages;
* Crossref records;
* Google Scholar records;
* Scopus or Web of Science records;
* uploaded source PDFs;
* library catalog records.

If a source cannot be checked externally, it must be marked as requiring manual verification.

## 3. The project does not replace human academic judgment

The project can help identify problems, but it cannot replace:

* the author;
* the supervisor;
* the dissertation council;
* journal editors;
* peer reviewers;
* field experts;
* university rules;
* publisher requirements.

The author remains responsible for the final manuscript.

## 4. The project does not create research

The project can help organize, review, and revise research writing.

It cannot replace:

* research design;
* source selection;
* data collection;
* fieldwork;
* archival work;
* case selection;
* theoretical reasoning;
* interpretation;
* disciplinary expertise.

A manuscript with weak research cannot be fixed by style alone.

## 5. The project does not make fabricated references acceptable

If a reference is fabricated, suspicious, or unverifiable, the correct solution is not to polish it.

The correct solution is to:

1. verify the source;
2. correct the bibliographic data if the source is real;
3. remove the reference if it is fake or unverifiable;
4. replace it only with a real source found through legitimate research;
5. revise claims that lack evidence.

The project must not invent replacement sources.

## 6. The project cannot reliably detect all plagiarism

The project is not a plagiarism detection system.

It can warn against copying and close imitation, but it cannot reliably compare a manuscript against all published literature.

Users should use appropriate plagiarism detection tools when required by:

* universities;
* journals;
* dissertation councils;
* publishers;
* funding bodies.

## 7. The project does not provide legal advice

The copyright and ethics notes in this repository are practical safety guidance.

They are not legal advice.

Copyright rules may vary by country, publisher, database, university, and license type.

Users are responsible for checking the legal conditions under which they access and use journal articles.

## 8. Journal style analysis has limits

The project can extract high-level writing patterns from target journal articles.

It can analyze:

* structure;
* abstract pattern;
* introduction logic;
* methodology language;
* argumentation model;
* citation behavior;
* conclusion structure;
* common academic formulas.

It should not be used to:

* copy another author's wording;
* imitate an individual author's distinctive style;
* reproduce article paragraphs;
* hide plagiarism;
* generate derivative text from one specific article.

A journal style profile is an analytical guide, not a permission to copy.

## 9. Hostile peer review is a simulation

The hostile peer review module is designed to stress-test a manuscript.

It can identify possible reviewer objections.

It cannot predict the actual decision of a real journal.

Different reviewers may evaluate the same manuscript differently.

A severe simulated review should be treated as a revision tool, not as final truth.

## 10. Rewriting has limits

The argument-forward rewriter can improve clarity, structure, and academic style.

It cannot solve problems that require new research.

Rewriting cannot replace:

* missing evidence;
* unverifiable references;
* weak method;
* unclear case selection;
* absent novelty;
* unsupported conclusions;
* inaccurate data;
* false claims.

If the manuscript has structural research problems, rewriting should come after correction of the research design and evidence base.

## 11. AI output may contain errors

AI-generated analysis or rewritten text may contain:

* factual mistakes;
* overconfident claims;
* inaccurate terminology;
* false assumptions;
* missing context;
* invented connections;
* unsuitable wording;
* incorrect interpretation.

All AI output must be checked by the author.

## 12. Field-specific limitations

The project is especially oriented toward Russian academic writing and ВАК-style humanities or social science manuscripts.

It may need adaptation for:

* natural sciences;
* medicine;
* engineering;
* law;
* quantitative social science;
* experimental studies;
* statistical modeling;
* laboratory research.

Different fields require different standards of evidence, method, structure, and terminology.

## 13. Responsible use rule

Use this project as a stress-test system.

Do not use it to bypass academic responsibility.

Before submission, check:

1. The research is real.
2. The sources are real.
3. The claims are supported.
4. The method fits the question.
5. The manuscript follows the journal rules.
6. The author understands and can defend every part of the text.
