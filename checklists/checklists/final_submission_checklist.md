# Final Submission Checklist

This checklist is used before submitting a Russian-language academic manuscript to a ВАК journal or related scholarly publication.

It should be used only after drafting, reference auditing, hostile peer review, structural reconstruction, and final coherence polishing.

## Core rule

Do not submit a manuscript only because the language looks polished.

Submit only when the research logic, evidence, references, structure, and journal requirements are all ready.

## 1. Target journal check

Before submission, confirm the target journal's official requirements.

Check:

* journal title;
* ВАК status or indexing status;
* accepted specialties;
* article length;
* abstract length;
* keyword number;
* citation style;
* bibliography format;
* file format;
* author information format;
* originality requirements;
* peer review procedure;
* submission system;
* publication ethics policy;
* AI-use policy, if available.

Status:

```text
[ ] Checked
[ ] Not checked
[ ] Requires author verification
```

Required action:

Do not rely only on imitation of published articles. Official journal guidelines control final formatting.

## 2. Article type check

Confirm the manuscript matches the expected article type.

Check:

* research article;
* review article;
* theoretical article;
* methodological article;
* case study;
* interdisciplinary article.

Questions:

* Does the manuscript match the target journal's accepted article types?
* Does the article structure fit this type?
* Does the article promise more than this type can deliver?

Status:

```text
[ ] Acceptable
[ ] Needs revision
[ ] Not suitable for this journal
```

## 3. Title check

A strong title should be specific and academically clear.

Check:

* title identifies the research object;
* title identifies the analytical focus;
* title is not too broad;
* title does not make unsupported causal claims;
* title matches the actual manuscript;
* title matches the target journal style.

Avoid:

```text
Traditional Culture and Active Longevity
```

Prefer:

```text
Transformation of Traditional Chinese Cultural Elements in Contemporary Architectural Environments Associated with Active Longevity Practices
```

Status:

```text
[ ] Ready
[ ] Needs narrowing
[ ] Needs journal-style adjustment
```

## 4. Abstract check

The abstract should contain:

* relevance;
* research gap;
* aim;
* material;
* methods;
* main result;
* contribution.

Required abstract logic:

```text
Актуальность
↓
Research gap
↓
Aim
↓
Material and method
↓
Result
↓
Contribution
```

Check:

* abstract does not introduce claims absent from the article;
* abstract does not exaggerate results;
* abstract includes material and method;
* abstract matches the conclusion;
* abstract length follows journal requirements.

Status:

```text
[ ] Ready
[ ] Needs revision
[ ] Rewrite after final article revision
```

## 5. Keywords check

Check:

* number of keywords matches journal requirements;
* keywords reflect the main topic;
* keywords include key concepts;
* keywords include field-specific terms;
* keywords are not too general;
* keywords are not repeated mechanically from the title only.

Recommended keyword types:

* object;
* subject;
* method;
* field;
* central concept;
* cultural or disciplinary context.

Status:

```text
[ ] Ready
[ ] Needs revision
```

## 6. Introduction check

The introduction should contain:

1. specific relevance;
2. literature context;
3. research gap;
4. object;
5. subject;
6. aim;
7. tasks;
8. material;
9. methods;
10. transition to analysis.

Check:

* relevance is specific;
* research gap is clear;
* object and subject are not confused;
* aim follows from the research gap;
* tasks correspond to article sections;
* material is named;
* methods are connected to the research question;
* no empty formulas are used.

Avoid:

```text
Актуальность темы не вызывает сомнений.
```

Prefer:

```text
Актуальность исследования обусловлена необходимостью анализа [specific problem].
```

Status:

```text
[ ] Ready
[ ] Needs structural revision
[ ] Needs full rewrite
```

## 7. Object, subject, aim, and tasks check

Check the logic:

| Element | Question                                        |
| ------- | ----------------------------------------------- |
| Object  | What broader phenomenon is studied?             |
| Subject | What specific aspect of the object is analyzed? |
| Aim     | What does the article seek to establish?        |
| Tasks   | What steps lead to the aim?                     |

Rules:

* object must be broader than subject;
* subject must be analyzable through the method;
* aim must be achievable in one article;
* tasks must not be decorative;
* each task must be completed in the article.

Status:

```text
[ ] Consistent
[ ] Partially consistent
[ ] Needs revision
```

## 8. Methodology check

The methodology must explain how the article answers the research question.

Check:

* methods are named;
* methods are explained;
* each method has a function;
* methods match the material;
* methods match the claim type;
* analytical procedure is described;
* methods are actually used in the analysis.

Weak:

```text
The article uses comparative, semiotic, and historical-cultural methods.
```

Stronger:

```text
Comparative analysis is used to identify differences between traditional and contemporary architectural forms; semiotic analysis makes it possible to interpret cultural elements as signs; historical-cultural analysis connects these signs to cultural memory and continuity.
```

Status:

```text
[ ] Ready
[ ] Method listed but not explained
[ ] Method does not match claims
[ ] Needs rewrite
```

## 9. Material check

The manuscript must clearly state what is being analyzed.

Check:

* material is named;
* selection criteria are explained;
* material is sufficient for the claim;
* material matches the method;
* examples are not merely decorative;
* case studies are justified;
* missing material is marked.

Use marker if needed:

```text
[AUTHOR MUST PROVIDE MATERIAL]
```

Status:

```text
[ ] Ready
[ ] Material incomplete
[ ] Selection criteria missing
[ ] Requires author verification
```

## 10. Literature review check

The literature review should synthesize sources.

Check:

* sources are grouped by approach;
* literature review creates the research gap;
* sources are relevant to the article;
* no fake or unverifiable sources remain;
* citations are not decorative;
* literature review is not a list of author names.

Weak:

```text
Ivanov studied X. Petrov studied Y. Sidorov studied Z.
```

Stronger:

```text
Existing research can be divided into three directions: first..., second..., third.... However, these approaches do not fully explain...
```

Status:

```text
[ ] Ready
[ ] Needs synthesis
[ ] Needs source verification
```

## 11. Main analysis check

The main analysis should apply the method to the material.

Each analytical unit should follow:

```text
claim
↓
material
↓
method application
↓
interpretation
↓
connection to research question
```

Check:

* analysis is not only description;
* each example is interpreted;
* evidence supports the claim;
* method is visible in the analysis;
* section order is logical;
* conclusions are not stated before evidence;
* each paragraph has a clear task.

Status:

```text
[ ] Ready
[ ] Descriptive but not analytical
[ ] Evidence missing
[ ] Needs reconstruction
```

## 12. Citation integrity check

Before submission, verify every reference.

Check:

* every source is real;
* every DOI is correct;
* every in-text citation has a bibliography entry;
* every bibliography entry is cited;
* every citation supports the attached claim;
* no decorative citations remain;
* no fake or AI-generated references remain;
* no unverifiable source remains unmarked.

Markers:

```text
[VERIFY REFERENCE]
[VERIFY DOI]
[UNVERIFIABLE SOURCE]
[SOURCE REQUIRED]
[POSSIBLE FABRICATED REFERENCE]
```

Status:

```text
[ ] Reference audit passed
[ ] Manual verification required
[ ] Do not submit
```

## 13. Citation-claim alignment check

For each cited claim, ask:

```text
Does this source actually support this exact claim?
```

Classify each citation:

* directly supported;
* partially supported;
* weakly supported;
* unsupported;
* source unavailable;
* citation mismatch.

Required action:

* narrow unsupported claims;
* replace unsuitable sources with verified relevant sources;
* remove decorative citations;
* remove claims without evidence.

Status:

```text
[ ] Passed
[ ] Problems remain
[ ] Full citation-claim audit required
```

## 14. High-risk claim check

Identify high-risk claims.

High-risk claim types:

* causal claims;
* health claims;
* active longevity claims;
* broad cultural claims;
* universal historical claims;
* claims about all architecture or all society;
* claims based on narrow material.

Check:

* does the manuscript make direct health-effect claims?
* does it claim architecture improves longevity?
* does it claim cultural symbols cause social or biological effects?
* does it use one case to prove a general rule?

If evidence is insufficient, revise the claim.

Weak:

```text
Architecture ensures active longevity.
```

Stronger:

```text
Architectural environment may form conditions that support social interaction, mobility, orientation, and everyday practices associated with active longevity.
```

Status:

```text
[ ] No high-risk unsupported claims
[ ] Some claims require revision
[ ] Critical claims remain
```

## 15. Paragraph task check

Every paragraph should have one clear task.

Check each paragraph:

* What is this paragraph doing?
* Does it introduce a problem?
* Does it define a concept?
* Does it present evidence?
* Does it analyze evidence?
* Does it interpret a result?
* Does it transition to the next step?
* Does it repeat earlier material?
* Does it mix too many functions?

Rule:

If you cannot name the paragraph's task, the paragraph is not ready.

Status:

```text
[ ] Paragraphs are clear
[ ] Some paragraphs need merging or splitting
[ ] Paragraph structure needs full review
```

## 16. Russian academic style check

Check style:

* formal;
* clear;
* precise;
* non-colloquial;
* not artificially complicated;
* no empty academic decoration;
* no excessive defensive formulas;
* no unsupported universal statements.

Reduce:

* актуальность темы не вызывает сомнений;
* необходимо отметить;
* на сегодняшний день;
* очевидно;
* всем известно;
* имеет огромное значение;
* данная статья не ставит целью;
* мы не утверждаем, что.

Prefer:

* анализ показывает;
* исследование направлено на;
* полученные результаты позволяют;
* актуальность обусловлена;
* научная значимость заключается в.

Status:

```text
[ ] Ready
[ ] Needs style polishing
[ ] Needs major language revision
```

## 17. Defensive formula check

Do not overuse defensive writing.

Avoid:

```text
This article does not claim that...
This study does not attempt to...
We do not argue that...
Настоящая статья не ставит целью...
Мы не утверждаем, что...
Это не означает, что...
```

Prefer positive analytical framing:

```text
The article examines the issue through...
The analysis focuses on...
The manuscript interprets this relation through...
В статье данная проблема рассматривается через...
```

Status:

```text
[ ] No excessive defensive formulas
[ ] Some defensive formulas should be rewritten
[ ] Defensive writing weakens argument
```

## 18. Conclusion check

The conclusion should:

* return to the aim;
* summarize supported findings;
* answer the research question;
* state contribution;
* avoid new concepts;
* avoid new sources;
* avoid broad unsupported claims;
* avoid simply repeating the abstract.

Check:

* does the conclusion match the analysis?
* does it exceed the evidence?
* does it introduce new material?
* does it clearly state what the article contributes?

Status:

```text
[ ] Ready
[ ] Needs narrowing
[ ] Needs full rewrite
```

## 19. Novelty and significance check

Novelty should be specific.

Weak:

```text
This topic has not been studied before.
```

Stronger:

```text
The novelty lies in interpreting traditional cultural elements not only as decorative forms, but as components of architectural environment connected to spatial practices.
```

Check:

* novelty follows from the results;
* novelty is not overstated;
* theoretical significance is specific;
* practical significance is justified;
* no contribution is claimed without evidence.

Status:

```text
[ ] Ready
[ ] Needs precision
[ ] Overstated
```

## 20. Formatting check

Check:

* title format;
* author information;
* affiliation;
* abstract formatting;
* keyword formatting;
* headings;
* tables;
* figures;
* captions;
* footnotes;
* reference style;
* line spacing;
* file format;
* anonymization, if required;
* supplementary materials, if required.

Status:

```text
[ ] Ready
[ ] Needs formatting
[ ] Journal guidelines must be checked
```

## 21. Ethics and copyright check

Check:

* no copyrighted full-text articles are included;
* no copied journal text is reused;
* no private data are exposed;
* no fake references are included;
* AI-generated text has been reviewed by the author;
* all claims are author-approved;
* journal AI-use policy is checked, if available.

Status:

```text
[ ] Passed
[ ] Requires author verification
[ ] Do not submit
```

## 23. Submission decision

Use one of the following decisions:

```text
Ready for submission
```

Use only if all major checks are passed.

```text
Ready after minor formatting corrections
```

Use if the research logic and references are ready, but formatting remains.

```text
Major revision required
```

Use if research logic, method, material, or conclusion still need revision.

```text
Do not submit
```

Use if references are unverifiable, claims exceed evidence, or the manuscript does not match the journal.

## Final rule

A manuscript is submission-ready only when the article's logic, evidence, citations, language, structure, and journal formatting all support the same argument.
